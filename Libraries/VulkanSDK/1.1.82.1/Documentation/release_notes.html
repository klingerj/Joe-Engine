Vulkan SDK for Windows Release Notes

Vulkan SDK 1.1.82.1 Release Notes

Version 1.1.82.1 for Windows
August 20, 2018

This SDK supports Vulkan API revision 1.1.82. The prior SDK supported Vulkan API revision 1.1.77.

Much of the documentation for this SDK is contained in the Getting Started Guide, found in the Documentation folder in this SDK and on the LunarXchange website.

Drivers: LunarG recommends that you update your Vulkan-capable hardware drivers to the latest recommended version from your hardware vendor. A list of Known Issues appears at the end of this document with more information.

Overview of changes in SDK 1.1.82.1

  • Fixed a bug where SPIR-V validation could incorrectly report valid SPIR-V as invalid. See the github issue for details.
  • Fixed incorrect sample vk_layer_settings.txt file in Windows SDK.

Overview of new features in SDK 1.1.82

  • Bug fixes (see the SDK Components section below)
  • Improved validation coverage
  • The standard Khronos validation layers have been migrated to use the canonical Vulkan VUID values, which are strings in the form of "VUID-vkCmdBlitImage-filter-00236". In doing so, the old integer-based enumerated values have been phased out and are no longer returned by log_msg() calls.
    • Note: Developers keying on these returned values for their development should use the map in the vk_validation_error_messages.h file to obtain the matching Vulkan VUIDs when performing their updates. This file can be found in the the SDK in the VulkanSDK\<SDK version>\Source\layers\ directory and in the layers directory of the Vulkan-ValidationLayers GitHub repository.

Known Issues

  • Applications, such as vulkaninfo or via, may produce a warning to inform the user the VK_LAYER_RENDERDOC_Capture layer is using the deprecated vkGetDeviceProcAddr tag. This tag was deprecated starting with JSON file version 1.1.0. The presentation of this warning can manifest in a variety of ways. This issue has been reported to the RenderDoc team.
  • SDK applications might not appear in the Start Menu if another version of the SDK is currently installed on the system. This can be corrected by uninstalling all other versions of the SDK.
  • Prior to the August 3, 2018 Steam update, applications such as cube and cubepp could experience a crash in VkCreateSwapChainKHR() if the Steam Overlay Layer (VK_LAYER_VALVE_steam_overlay) was enabled. If this error occurs, first try updating your Steam client by clicking the "Steam" menu and selecting "Check for Steam Client Updates...". If you are unable to update the Steam client or the error persists, you can avoid the behavior by disabling the Steam Overlay Layer in one of the following ways:
    1. Set the DISABLE_VK_LAYER_VALVE_steam_overlay_1 environment variable equal to 1. This can be accomplished by editing the user environment variables under the Advanced System Properties menu or if you are using a command prompt to execute the application, run the following command from the command prompt (Note: This change will only persist for a single command prompt session):
    export DISABLE_VK_LAYER_VALVE_steam_overlay_1=1
    1. Edit the Windows registry. To open the Windows registry, navigate to the Start Menu and either in the Run box or the Search box type regedit and press Enter. Navigate to Computer -> HKEY_LOCAL_MACHINE_SOFTWARE -> SOFTWARE -> Khronos -> Vulkan -> ImplicitLayers. Here you should find the SteamOverlayVulkanLayer64.json entry. Double click the file name and change the value in the "Value Data" box to 1. If you are running a 64-bit version of Windows you will have to perform an additional step by making the same change in the WOW6432Node directory. The registry entries found in the WOW6432Node directory are referenced when 32-bit programs are run on a 64-bit system. To make this change, navigate to Computer -> HKEY_LOCAL_MACHINE_SOFTWARE -> SOFTWARE -> WOW6432Node -> Khronos -> Vulkan -> ImplicitLayers. For the SteamOverlayVulkanLayer.json entry, double-click the file name and change the value in the "Value Data" box to 1.
